# Техническое задание
## Модуль «Чек-лист готовности склада к внедрению WMS»

**Версия:** 1.0  
**Дата:** апрель 2026  
**Продукт:** Сервис самоаудита склада  
**Тип документа:** Техническое задание на разработку

---

## 1. Назначение модуля

Модуль является вторым контентным продуктом сервиса самоаудита склада (в дополнение к основному опроснику из 20 вопросов). Цель — дать руководителям честную самооценку готовности к внедрению WMS **до** переговоров с интеграторами.

**Ценность для пользователя:**
- Быстро понять, есть ли технические или организационные блокеры для WMS
- Получить практические советы по устранению пробелов
- Сформировать аргументы для разговора с директором или интегратором

**Место в воронке:** лид-магнит (бесплатный PDF) → сбор email → прогрев → предложение полного аудита.

---

## 2. Пользовательский сценарий

```
Пользователь приходит на лендинг
    → видит CTA «Скачать чек-лист WMS»
    → вводит email (форма)
    → получает PDF на почту + видит интерактивную версию в браузере
    → проходит 10 пунктов (отмечает статус каждого)
    → видит итоговый результат с рекомендацией
    → получает CTA на основной самоаудит / запись на аудит
```

## 3. Функциональные требования

### 3.1 Форма сбора email (gate перед чек-листом)

| Поле | Тип | Обязательное | Валидация |
|---|---|---|---|
| Email | text | да | формат email, уникальность не проверяется |
| Имя | text | нет | — |

- После отправки формы: показать интерактивный чек-лист + отправить PDF на email
- Кнопка: «Получить чек-лист →»
- Под формой: «Без спама. Только чек-лист и, возможно, одно письмо с полезным контентом.»

### 3.2 Интерактивный чек-лист (веб-версия)

**Структура одного пункта:**

Каждый из 10 пунктов содержит:
1. **Номер + заголовок** (визуально выделен, тёмный фон)
2. **Подзаголовок** — одно предложение, объясняющее, почему пункт важен
3. **Две колонки:** «✅ Готово к WMS» / «🚫 Ещё не готово» — конкретное описание каждого состояния
4. **Блок предупреждения/совета** (опционально) — выделен цветом (оранжевый левый бордер)
5. **Переключатель статуса** — три варианта: `✅ Готово` / `⚙️ В процессе` / `🚫 Не готово`

**Поведение переключателя:**
- По умолчанию — ни один вариант не выбран (серый)
- При выборе варианта — пункт визуально меняет состояние (зелёный / жёлтый / красный рамка)
- Прогресс-бар вверху страницы обновляется в реальном времени

**Прогресс-бар:**
- Показывает количество заполненных пунктов из 10
- Текст: «Заполнено: N / 10»

### 3.3 Блок итогов (показывается после заполнения всех 10 пунктов или по кнопке «Показать результат»)

**Логика подсчёта:**
- Считаются только пункты со статусом `✅ Готово`
- Пункты `⚙️ В процессе` не засчитываются в балл, но помечаются отдельно

**Итоговые уровни:**

| Баллов (✅) | Статус | Сообщение |
|---|---|---|
| 0–3 | 🔴 Ещё рано | «Сначала наведите базовый порядок: адресация, учёт, разметка. Вернитесь к чек-листу через 3–6 месяцев.» |
| 4–6 | 🟡 Частичная готовность | «Есть серьёзные пробелы. Определите 2–3 блокера и устраните их за 1–2 месяца до старта проекта.» |
| 7–8 | 🟢 Готовы с оговорками | «Можно начинать переговоры с поставщиками WMS. Учтите незакрытые пункты в плане внедрения.» |
| 9–10 | ✅ Полная готовность | «Вы готовы. Начинайте тендер WMS. Основные риски закрыты.» |

**Дополнительно в блоке итогов:**
- Список незакрытых пунктов (`🚫 Не готово`) с кратким напоминанием
- Список пунктов `⚙️ В процессе` отдельным блоком
- CTA: «Пройти самоаудит склада за 15 минут →» (ссылка на основной модуль)
- CTA: «Записаться на консультацию» (ссылка/форма)

### 3.4 Генерация и отправка PDF

- PDF генерируется на сервере по шаблону (не на клиенте)
- В PDF подставляются статусы пользователя (если он прошёл чек-лист перед скачиванием) **или** отправляется шаблонный PDF без статусов, если email собран до прохождения
- PDF содержит все 10 пунктов, итоговую таблицу, советы и CTA

**Письмо:**
- Тема: «Ваш чек-лист готовности склада к WMS»
- Тело: 3–4 предложения + кнопка «Открыть чек-лист» (ссылка на веб-версию с сохранённым прогрессом)
- Отправитель: настраивается в CMS/конфиге

## 4. Содержание чек-листа (10 пунктов)

### Пункт 1. Адресное хранение
**Подзаголовок:** Каждая ячейка, стеллаж и зона имеют уникальный адрес. Без этого WMS просто некуда записывать остатки.

- **✅ Готово:** Каждое место хранения имеет уникальный код. Система знает, что где лежит.
- **🚫 Не готово:** Товар хранится «там где место». Кладовщик знает по памяти — система нет.
- **Предупреждение:** Это единственный нефинансовый блокер: без адресного хранения WMS физически невозможна.
- **Совет:** Начните с разметки пола и нумерации стеллажей. Это 1–2 рабочих дня и 5–10 тыс. руб.

### Пункт 2. Стабильность номенклатуры
**Подзаголовок:** WMS тяжело работает с постоянно меняющимися SKU — каждый новый товар требует настройки правил хранения.

- **✅ Готово:** Номенклатура стабильна или меняется предсказуемо (сезонность, коллекции).
- **🚫 Не готово:** Каждую неделю появляется 20–50 новых SKU без регламента.
- **Предупреждение:** Частые неконтролируемые изменения номенклатуры — не приговор, но требуют отдельного регламента в WMS.
- **Совет:** Введите процедуру заведения нового SKU до внедрения WMS — это упростит настройку системы.

### Пункт 3. Точность текущего учёта
**Подзаголовок:** Если данные в учётной системе расходятся с реальностью — WMS унаследует этот хаос и усилит его.

- **✅ Готово:** Расхождения при инвентаризации — менее 1–2%. Данные в системе актуальны.
- **🚫 Не готово:** Инвентаризация регулярно выявляет расхождения >5%. «По системе есть, по факту нет».
- **Предупреждение:** WMS не исправляет плохой учёт — она работает с теми данными, которые вы в неё загружаете.
- **Совет:** Перед внедрением проведите полную инвентаризацию и сверку. Это обязательная точка старта.

### Пункт 4. Уровень ошибок при отборке
**Подзаголовок:** WMS даёт максимальный эффект, когда есть что улучшать. Критический порог — 2% и выше.

- **✅ Готово:** Ошибки при комплектации — 2% и более. Есть явная проблема, которую WMS решает.
- **🚫 Не готово:** Ошибок почти нет (< 0,5%). WMS избыточна по стоимости для вашего объёма.
- **Предупреждение:** Если у вас уже 0,1% ошибок при 200 строках в день — ROI от WMS будет очень долгим.
- **Совет:** Посчитайте стоимость одной ошибки (возврат + повторная доставка + претензия клиента). Это аргумент для бюджета.

### Пункт 5. Персонал: готовность к изменениям
**Подзаголовок:** Самый недооценённый риск. WMS меняет работу каждого кладовщика — и провоцирует сопротивление.

- **✅ Готово:** Команда открыта к изменениям. Руководство поддерживает проект публично.
- **🚫 Не готово:** Сотрудники говорят «и так работает». Руководитель склада против или нейтрален.
- **Предупреждение:** 80% неудачных внедрений WMS — это проблема людей, а не технологий.
- **Совет:** Назначьте «чемпиона изменений» среди старших кладовщиков. Их поддержка важнее поддержки IT.

### Пункт 6. Штрихкодирование товаров
**Подзаголовок:** WMS со сканерами требует штрихкодов на каждой единице. Без них отборка по-прежнему будет бумажной.

- **✅ Готово:** 80–100% позиций имеют штрихкоды (EAN, QR, Code128). Сканеры или ТСД уже есть.
- **🚫 Не готово:** Значительная часть товаров без штрихкодов. Этикетировать нужно самим.
- **Совет:** Если поставщики не клеят штрихкоды — заложите в проект WMS зону приёмки с этикетировочным принтером.

### Пункт 7. Интеграция с учётной системой (1С, ERP)
**Подзаголовок:** WMS — не изолированный остров. Она должна получать заказы и отдавать подтверждения в вашу основную систему.

- **✅ Готово:** Есть 1С или ERP с открытым API или готовым коннектором для выбранной WMS.
- **🚫 Не готово:** Учёт ведётся в Excel или самописной системе без API.
- **Предупреждение:** Без интеграции WMS = второй ручной ввод данных. Это хуже, чем до автоматизации.
- **Совет:** Спросите у поставщика WMS: «Какие коннекторы к 1С у вас есть из коробки?» Это сэкономит месяцы разработки.

### Пункт 8. Техническая инфраструктура
**Подзаголовок:** Wi-Fi на каждом метре склада, серверная мощность или облако, ТСД-устройства для каждого кладовщика.

- **✅ Готово:** Стабильный Wi-Fi покрывает весь склад. Есть бюджет на ТСД (3–8 тыс. руб./мес. аренда или 40–80 тыс. покупка).
- **🚫 Не готово:** Wi-Fi только в офисе. Нет устройств для кладовщиков. ИТ-поддержки нет.
- **Совет:** Аренда ТСД у поставщика WMS часто выгоднее покупки на старте — меньше рисков при смене системы.

### Пункт 9. Бюджет: лицензии, железо, внедрение
**Подзаголовок:** Честный тест. WMS — не бесплатный плагин. Полная стоимость включает лицензии, интеграцию, обучение и поддержку.

- **✅ Готово:** Есть бюджет от 300–500 тыс. руб. на старт (облачная WMS для 2–5 пользователей + интеграция).
- **🚫 Не готово:** Бюджет «до 100 тыс. руб.» или «посмотрим по ходу». Проект не взлетит.
- **Предупреждение:** «Бесплатные» WMS — это лицензия без поддержки и интеграции. Итоговая стоимость владения та же.
- **Совет:** Закладывайте ×1,5–2 от стоимости лицензии на интеграцию и обучение. Это реалистичный бюджет.

### Пункт 10. Спонсор проекта на уровне директора
**Подзаголовок:** WMS затрагивает склад, IT, бухгалтерию и коммерцию одновременно. Без высокого спонсора проект гибнет в согласованиях.

- **✅ Готово:** Генеральный или операционный директор лично заинтересован и готов разрешать конфликты между отделами.
- **🚫 Не готово:** Проект инициирован начальником склада или логистом без поддержки сверху.
- **Предупреждение:** Внедрение WMS — это организационный проект, а не IT-проект. Без топ-менеджмента он не завершится.
- **Совет:** Сформулируйте директору бизнес-кейс в рублях: «Мы теряем X руб./мес., WMS окупается за Y месяцев». Это его язык.

## 5. Нефункциональные требования

### 5.1 Интерфейс и UX
- Адаптивная вёрстка: desktop / tablet / mobile
- Поддержка браузеров: Chrome 110+, Firefox 110+, Safari 16+, Edge 110+
- Сохранение прогресса в `localStorage` (пользователь может вернуться)
- Анимация при выборе статуса (плавная, не отвлекающая)
- Время загрузки страницы: < 2 сек (LCP)

### 5.2 Цветовая схема

| Элемент | Цвет (HEX) |
|---|---|
| Основной акцент (кнопки, номера пунктов) | `#E8610A` (оранжевый) |
| Тёмный фон заголовков | `#1A1A2E` |
| Статус «Готово» | `#27AE60` (зелёный) |
| Статус «Не готово» | `#C0392B` (красный) |
| Статус «В процессе» | `#F39C12` (жёлтый) |
| Фон предупреждения | `#FFF3E6` с левым бордером `#E8610A` |

### 5.3 Типографика
- Основной шрифт: Inter или Arial (системный фоллбэк)
- Заголовки пунктов: 18–20px, bold
- Основной текст: 14–16px, regular
- Подсказки/советы: 14px, italic

## 8. PDF-шаблон

### 8.1 Структура PDF (A4, портрет)

1. **Титульная область** — тёмный фон, название документа, подзаголовок, дата генерации
2. **Вводный блок** — «Зачем этот чек-лист?» (1–2 предложения)
3. **10 пунктов** — каждый на отдельном блоке: заголовок / подзаголовок / «Готово» + «Не готово» / совет
4. **Поле статуса** — если пользователь прошёл чек-лист, в PDF подставляется его выбор (иконка + цвет)
5. **Итоговая таблица** — 4 уровня с пояснениями
6. **Футер** — копирайт + CTA + ссылка на самоаудит

### 8.2 Технология генерации
- Серверная генерация: **Puppeteer** (HTML → PDF) или **PDFKit**
- Шаблон: HTML + CSS (Tailwind или inline-стили)
- Формат на выходе: PDF/A, без пароля, размер < 2 МБ

## 9. Требования к безопасности и конфиденциальности

- Email хранится в зашифрованном виде
